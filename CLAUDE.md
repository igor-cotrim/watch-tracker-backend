## Architecture

Express 5 + TypeScript backend with Drizzle ORM (PostgreSQL via Supabase) and TMDB as the external media data source.

### Route Groups (mounted under `/api/`)

- **`/watchlist`** â€” CRUD for user watchlists. All routes require auth. Supports filtering by `status` and `media_type` query params.
- **`/media`** â€” Media details from TMDB (public), plus user ratings and episode watch tracking (auth required).
- **`/discover`** â€” Trending, search, now-playing (Brazil region), filtered discovery. All public.
- **`/profile`** â€” User profile stats (movies watched, shows tracking, episodes watched). All routes require auth.
- **`/health`** â€” Health check endpoint (defined in `src/app.ts`).

### Auth Pattern

Auth middleware (`src/middleware/auth.ts`) uses the Supabase **service role key** to verify bearer tokens server-side via `supabase.auth.getUser(token)`. Verified user is attached as `req.user` on `AuthenticatedRequest` (defined in `src/types/index.ts`). Routes opt-in to auth by applying the middleware.

### Database (Drizzle ORM)

Schema in `src/db/schema.ts` defines four tables: `profiles`, `userWatchlist`, `userEpisodesWatched`, `userRatings`. All user tables reference `profiles.id` (UUID from Supabase Auth).

Migrations live in `drizzle/` â€” generated by `drizzle-kit generate`, applied by `drizzle-kit migrate`. The `db` singleton is exported from `src/db/index.ts`.

### TMDB Service

`src/services/tmdb.ts` exposes typed methods (`getMediaDetails`, `getTrending`, `search`, `discover`, `getNowPlaying`, `getSeasonDetails`) backed by an axios instance with the API key injected as a default query param.

### Validation

Zod schemas are defined inline in route files. Request bodies are validated with `safeParse()`, returning 400 with issue details on failure.

### Error Handling

Route handlers use try-catch with `next(error)`. The global error handler (`src/middleware/errorHandler.ts`) returns stack traces in development (`NODE_ENV !== 'production'`) and generic messages in production.
